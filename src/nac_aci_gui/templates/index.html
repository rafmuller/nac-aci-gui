Welcome to NaC Workflows

Network as Code - Manage your network infrastructure using Infrastructure as Code

The intent of this tool is to demonstrate what is possible with NaC-API. NaC API is a north bound API gateway to Source Code Managers. The goal of NaC-API is to provide a simple RESTful and GraphQL API interface to interact with Network as Code data models stored in Source Code Managers (SCM) such as GitHub and GitLab. This allows for easy integration with ITSM systems or other automation tools ( or self developed portals ).

By providing this capability, NaC-API enables organizations to front-end the infrastructure as code models with custom applications or ITSM for simplification of network operations. Yet by leveraging NaC-API, you still retain the full power of pre-change and post-change validation when performing network changes. In addition, the SCM remains the source of truth for all network with powerful version control capabilities.

Create and manage network configurations

Create Networks

The Network as Code API (NaC-API) provides seamless integration with NetBox IPAM to automate network provisioning workflows. When triggered through the Actions Network interface, NaC-API connects directly to the NetBox API to retrieve essential network data including VLANs and IP prefixes. This automated data retrieval eliminates manual configuration errors and ensures consistency between your IPAM system and network infrastructure deployments.

Once the data is retrieved from NetBox, NaC-API processes and transforms the VLAN and prefix information into a standardized network schema format. This generated YAML schema is then automatically committed to your configured Source Code Management (SCM) repository via a merge request, enabling full GitOps workflow capabilities. This integration streamlines the network provisioning process by bridging IPAM data with infrastructure-as-code practices, providing complete audit trails and version control for all network configuration changes.
